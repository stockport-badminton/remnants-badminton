## Commands

```bash
npm run dev      # Dev server at http://localhost:4321
npm run build    # Production build → dist/
npm run preview  # Preview built dist/ locally
npm run scrape   # Manually fetch latest fixtures from Stockport & Crewe
```

No test suite or linter is configured.

## Architecture

Static site for a badminton club, built with **Astro 4** (SSG mode), styled with **Tailwind CSS**, hosted on **Netlify**, and content-managed via **Decap CMS**.

### Data sources

All data is static files — no database:

- `src/data/theme.json` — site colors, name, tagline (CMS-editable, injected as CSS custom properties in `src/layouts/Base.astro`)
- `src/data/home.json` — hero and about text (CMS-editable)
- `src/data/fixtures.json` — fixture/results data, **auto-generated by the scraper** (do not hand-edit)
- `src/content/members/` — one markdown file per member, managed via Decap CMS, validated by the Zod schema in `src/content/config.ts`

### Fixture scraping pipeline

`scripts/scrape-fixtures.mjs` pulls from two sources:
- **Stockport**: JSON API at `stockport-badminton.co.uk` (pre-filtered for Remnants)
- **Crewe**: HTML table scrape via Cheerio at `crewebadminton.org.uk`

It merges both into `src/data/fixtures.json`. GitHub Actions (`.github/workflows/scrape-fixtures.yml`) runs this every 6 hours and auto-commits if the file changed, which triggers a Netlify rebuild.

The scraper infers the year for date strings that omit it (badminton season: Sep–May). If one source fails, existing data for that source is preserved.

### CMS

Editors log in at `/admin/` via Netlify Identity. Decap CMS (Git Gateway) commits changes directly to the repo. The CMS manages three collections: Members, Home Page content, and Theme & Branding.

### Theming

Colors are defined in `src/data/theme.json` and injected as inline CSS custom properties (`--color-primary`, `--color-secondary`, `--color-accent`) in `src/layouts/Base.astro`. Tailwind extends those via `tailwind.config.mjs` so `text-primary`, `bg-secondary`, etc. work throughout templates.

### Client-side interactivity

Vanilla JS only — no framework:
- Fixture page (`src/pages/fixtures.astro`): upcoming/results toggle and league filter (Stockport/Crewe/All)
- Navigation (`src/components/Nav.astro`): mobile hamburger menu toggle
- Contact form (`src/pages/contact.astro`): Netlify Forms (no backend needed)
